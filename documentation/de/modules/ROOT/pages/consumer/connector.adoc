= Definition eines Consumer Connectors

Das link:./{attachmentsdir}/javadoc/io/transconnect/connector/api/IConnector.html[`IConnector`,window=_blank] ist der Hauptbestandteil des SDK.
Es ist verantwortlich für die Herstellung der Verbindung zum externen System.
Außerdem ist es die Klasse, die die vom Benutzer einstellbaren Konfigurationseigenschaften sowie die ausführbaren Interaktionen definiert.

== Erstellen der Connector-Klasse

Die Connector-Klasse kann neben der Connection-Klasse aus dem vorherigen Abschnitt erstellt werden.
Wir nennen sie `CountCharactersConnector` und sie implementiert das Interface link:./{attachmentsdir}/javadoc/io/transconnect/connector/api/consumer/ConsumerConnector.html[`ConsumerConnector`,window=_blank].

[source,java,indent=0]
....
@Conntector
public final class CountCharactersConnector implements ConsumerConnector {
    // methods to implement
}
....

Die Annotation link:./{attachmentsdir}/javadoc/io/transconnect/connector/api/extension/Connector.html[`@Connector`,window=_blank] wird verwendet, um die Klasse als Connector-Implementierung zu kennzeichnen.
Später kann sie verwendet werden, um xref:extensions/introduction.adoc[Erweiterungen (extensions)] zu deklarieren, die vom Connector genutzt werden.

Er muss Informationen über sich selbst bereitstellen.

Zum Beispiel:

* den Namen des Connectors
* die Version
* die Konfigurationseigenschaften
* die verfügbaren Interaktionen

== Unterschiede zwischen Consumer und Producer Connectors

Es gibt einige wesentliche Unterschiede zwischen Consumer und Producer Connectors:

1. **Interaktionen (Interactions)**: Consumer Connectors definieren Interaktionen, die ausgeführt werden können und sowohl das Eingabe- als auch das Ausgabe-Nachrichtenformat spezifizieren.
Producer Connectors geben in der Regel nur das Ausgabe-Nachrichtenformat an.

2. **Ablauf der Ausführung (Execution Flow)**: Consumer Connectors empfangen Nachrichten und erzeugen Antworten, während Producer Connectors proaktiv Nachrichten generieren.

3. **Verbindungsmanagement (Connection Management)**: Consumer Connections konzentrieren sich darauf, eingehende Anfragen effizient zu verarbeiten, während Producer Connections den Lebenszyklus der ausgehenden Nachrichtenerzeugung verwalten müssen.
