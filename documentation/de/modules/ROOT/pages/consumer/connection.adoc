= Aufbau einer Consumer-Verbindung

Eine Connector-Implementierung besteht aus zwei Teilen: der Connection und dem Connector selbst.
Die Connection-Klasse enthält die Geschäftslogik, zum Beispiel stellt sie eine Verbindung zu einer Datenbank her oder verarbeitet eingehende Anfragen.
Je nachdem, ob ein Producer- oder Consumer-Connector implementiert wird, verarbeitet die Connection unterschiedliche Workflows.

In unserem Beispiel implementieren wir eine einfache Connection, die Zeichen in einer XML-Nachricht zählt.
Die Connection wird von der Connector-Klasse erstellt.
Die Connector-Klasse sehen wir uns im nächsten Abschnitt an.

Weitere Details zu den Klassenabhängigkeiten im Connector SDK finden Sie unter xref:architecture.adoc[Connector SDK Architektur].

== Erstellen der Connection-Klasse

Lassen Sie uns eine neue Java-Klasse erstellen und das link:./{attachmentsdir}/javadoc/io/transconnect/connector/api/consumer/ConsumerConnection.html[`ConsumerConnection`,window=_blank]-Interface implementieren.

. Erstellen Sie ein neues Package im Verzeichnis `src/main/java`.
+
Der Paketname sollte `com.example.connector.consumer` lauten.
. Nun erstellen wir eine neue Java-Klasse im Package mit dem Namen `CountCharactersConnection`.
. Als Nächstes fügen wir der Klasse das Interface link:./{attachmentsdir}/javadoc/io/transconnect/connector/api/consumer/ConsumerConnection.html[`ConsumerConnection`,window=_blank] hinzu.
+
Das ConsumerConnection-Interface ist Teil des Connector SDK und stellt die notwendigen Methoden zur Verfügung, um eine Connection zu erstellen.
+
Dazu ändern Sie die Klassendefinition wie folgt:
+
[source,java,indent=0]
....
public final class CountCharactersConnection implements ConsumerConnection {
    // methods to implement
}
....

== Implementierung der Connection-Methoden

Das ConsumerConnection-Interface erzwingt die Implementierung mehrerer Methoden.
Zusätzlich muss die Connection in der Lage sein, ein link:./{attachmentsdir}/javadoc/io/transconnect/connector/api/Context.html[`Context`,window=_blank]-Objekt übergeben zu bekommen.
Dieses enthält die Konfiguration, die zum Aufbau der Connection verwendet werden soll.

Beispielhafte Konfigurationseigenschaften sind:

* Host und Port eines externen Systems
* Verbindungs-Timeout
* Verarbeitungsoptionen wie z.B. ob Leerzeichen mitgezählt werden sollen

Der bequemste Weg, den Context an die Connection zu übergeben, ist über den Konstruktor.

[source,java,indent=0]
....

....

Diese Methode wird von TRANSCONNECT aufgerufen, um die Verbindung zu schließen.
Es ist eine gute Praxis, die Methode defensiv zu schreiben, sodass sie keine Ausnahmen verursachen kann.

TRANSCONNECT ruft die `close`-Methode unter anderem in folgenden Situationen auf:

* Beim Herunterfahren von TRANSCONNECT
* Wenn die Connection als inaktiv erkannt wird
* Wenn der Connector nicht mehr verwendet wird
* Wenn die Verbindungseigenschaften geändert wurden und die Connection neu aufgebaut werden muss
* Wenn ein unbehandelter Fehler in der Connection aufgetreten ist

== Vergleich mit Producer Connection

Im Gegensatz zu einer ProducerConnection, die aktiv Nachrichten erzeugt, ist eine ConsumerConnection passiv und reagiert auf eingehende Anfragen. Die wichtigsten Unterschiede sind:

1. ProducerConnection verwendet eine `connect`-Methode, die einen Listener erhält, um Nachrichten zurückzumelden, während die `connect`-Methode der ConsumerConnection keine Argumente erhält.

2. ConsumerConnection verfügt über eine `execute`-Methode zur Verarbeitung eingehender Nachrichten, die bei ProducerConnection nicht existiert.

3. ProducerConnection ist für das Erstellen und Senden von Nachrichten verantwortlich, während ConsumerConnection für die Verarbeitung eingehender Nachrichten und das Erstellen von Antworten zuständig ist.
