= Connector SDK Architektur

== Einführung

Das TRANSCONNECT Connector SDK bietet ein standardisiertes Framework zur Integration externer Systeme mit der TRANSCONNECT Plattform. Dieses Dokument beschreibt die Architektur, die Kernkomponenten und die Interaktionsmuster des SDK.

=== Zentrale Konzepte

Das SDK basiert auf zwei primären Connector-Typen:

[cols="1,4", options="header"]
|===
|Connector-Typ |Zweck
|*Consumer Connectors* |Empfangen und verarbeiten Nachrichten von TRANSCONNECT und interagieren mit externen Systemen zur Erfüllung von Anfragen
|*Producer Connectors* |Überwachen externe Systeme und erzeugen Nachrichten, die an TRANSCONNECT gesendet werden
|===

Beide Connector-Typen teilen eine gemeinsame Designphilosophie:

* Klare Trennung zwischen Connector und Connection
* Standardisierte Nachrichtenverarbeitung
* Konfigurationsgesteuertes Verhalten
* Lebenszyklusmanagement

== Best Practices

Beim Implementieren von Connectoren mit dem SDK sollten folgende Muster beachtet werden:

1. **Trennung der Verantwortlichkeiten (Separation of Concerns)**
* Connectoren erstellen Connections und stellen Metadaten bereit
* Connections enthalten die Geschäftslogik und übernehmen die Nachrichtenverarbeitung

2. **Fehlerbehandlung (Error Handling)**
* Verwenden Sie try-with-resources für eine ordnungsgemäße Ressourcenfreigabe
* Verpacken Sie externe Ausnahmen in link:{attachmentsdir}/javadoc/io/transconnect/connector/api/TransconnectConnectorException.html[`TransconnectConnectorException`,window=_blank]
* Protokollieren Sie aussagekräftige Fehlermeldungen

3. **Konfigurationsmanagement (Configuration Management)**
* Definieren Sie klare Eigenschaftsnamen mit hilfreichen Beschreibungen
* Stellen Sie sinnvolle Standardwerte bereit, wo möglich
* Validieren Sie die Konfiguration frühzeitig

4. **Connection-Lebenszyklus**
* Sorgen Sie für eine ordnungsgemäße Aufräumlogik in der Methode `close()`
* Implementieren Sie thread-sicheres Prüfen des Connection-Zustands
* Entwerfen Sie Connections, wenn möglich, zustandslos
