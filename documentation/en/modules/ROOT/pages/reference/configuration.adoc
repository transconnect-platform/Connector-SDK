= Configuration Reference

== Overview

TRANSCONNECT connectors use a standardized configuration model that defines their identity, capabilities, and configuration options.
This document outlines the key components and properties used when configuring connectors.

[TIP]
====
Understanding the connector configuration model is essential for both connector developers and integration specialists.
====

== Core Components

=== Connector Types

TRANSCONNECT supports two primary connector types:

Consumer Connectors::
Receive and process incoming messages through defined interactions.

Producer Connectors::
Generate messages according to schema definitions and process result messages.

== Configuration Elements

=== Connector Identity

[#connector-identity]
Defines the core identity and metadata of a connector.

[cols="2,5,1",options="header"]
|====
|Property |Description |Required
|specVersion |Version of the specification format (e.g., 1.0.0) |Yes
|type |Unique URN identifier (e.g., "urn:TRANSCONNECT:connector:sample-consumer") |Yes
|version |Connector implementation version |Yes
|vendor |Provider organization name |Yes
|displayName |UI display names with language variants |Yes
|description |Purpose descriptions with language variants |Yes
|labels |Categorization tags (e.g., ["sample", "greeting"]) |No
|====

[#properties]
=== Properties Configuration

==== Property Definitions

All connector properties share these common attributes:

[cols="2,5,1",options="header"]
|====
|Attribute |Description |Required
|id |Unique property identifier |Yes
|displayName |UI display names with language variants |Yes
|description |Purpose descriptions with language variants |Yes
|type |Data type (TEXT, SECRET, INTEGER, FLOAT, BOOLEAN, SELECT) |Yes
|defaultValue |Default value when none is provided |No
|required |Whether user input is mandatory |Yes
|validator |Input validation rules |No
|dependsOn |Property dependencies and constraints |No
|entries |For SELECT type - available options |Yes (for SELECT)
|====

==== Property Types

Each property must use one of the following data types:

[cols="1,2,4",options="header"]
|====
|Type |Java Class |Usage
|TEXT |String |General text input
|SECRET |String |Sensitive data (passwords, tokens, etc.)
|INTEGER |Integer |Whole numbers
|FLOAT |Double |Decimal numbers
|BOOLEAN |Boolean |True/false values
|SELECT |String |Selection from predefined options
|====

==== Localized Text Format

The localization system uses locale-specific messages:

[cols="2,5",options="header"]
|====
|Attribute |Purpose
|locale |Language code (e.g., "en", "de")
|message |Translated text
|====
